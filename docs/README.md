# Interview Docs Index

This folder contains concise, practical materials to prepare for system design and implementation interviews. Use the index below to navigate.

- implementation.md — pragmatic coding practices from basics to advanced
- design-principles.md — core design principles and architectural patterns
- scalability.md — strategies and patterns to scale systems reliably
- system-design-scenarios.md — walkthroughs of common interview problems
- interview-qa.md — question bank with clear, actionable answers
- system-design-template.md — reusable checklist and skeleton for any design prompt
- behavioral.md — STAR-based guide for behavioral interviews
- algorithms-cheatsheet.md — Big-O, patterns, and quick templates
- coding-patterns.md — practical coding templates for common problems
- security-checklist.md — security considerations and safeguards
- production-readiness.md — deployment, observability, reliability checklist
- estimation-examples.md — back-of-the-envelope examples and formulae
- architecture-diagrams.md — fast C4 diagramming guidance
- movieshelf-system-design.md — project-specific end-to-end design example
- testing-qa.md — testing strategy, commands, and QA checklist

## How to Use

1. Start with system-design-template.md to structure your approach.
2. Refresh fundamentals in implementation.md and design-principles.md.
3. Study scalability.md for production-grade considerations.
4. Practice with system-design-scenarios.md.
5. Review interview-qa.md for rapid recall.
6. Prepare stories with behavioral.md using the STAR method.
7. Warm up coding with algorithms-cheatsheet.md and coding-patterns.md.
8. Use estimation-examples.md and architecture-diagrams.md during whiteboard rounds.
9. Validate production concerns with security-checklist.md and production-readiness.md.
10. Tailor answers using movieshelf-system-design.md for concrete examples.
11. Cite testing-qa.md to demonstrate practical QA and test coverage.

## Typical Interview Flow

1. Clarify requirements and constraints.
2. Back-of-the-envelope estimations.
3. API design and data modeling.
4. High-level architecture and data flow.
5. Bottlenecks and scaling plan.
6. Reliability, consistency, and trade-offs.
7. Security, privacy, and observability.
8. Evolution plan and cost awareness.
