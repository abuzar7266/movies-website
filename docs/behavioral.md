# Behavioral Interview Guide

## Framework

- Use STAR: Situation, Task, Action, Result
- Add Reflection: what you learned, what you’d do differently
- Be specific: quantify impact; name stakeholders; show constraints

## Core Themes

- Ownership: end-to-end responsibility; proactive problem-solving
- Bias for Action: unblock quickly; iterate; manage risk
- Customer Focus: understand user pain; measure outcomes
- Communication: crisp updates; escalate early; align stakeholders
- Teamwork: mentorship; cross-team collaboration; resolve conflicts constructively
- Leadership: set vision; drive consensus; make trade-offs explicit

## Common Prompts (Use STAR)

- A time you handled an incident
- A time you disagreed and committed
- Delivering under ambiguous requirements
- Migrating legacy systems without downtime
- Driving a cross-functional initiative
- Mentoring or leveling up a teammate

## Answer Tips

- Frame constraints: time, budget, compliance, tech debt
- Call out trade-offs: performance vs simplicity; consistency vs availability
- Show mechanisms: runbooks, checklists, automation, metrics
- Quantify: latency reductions, cost savings, adoption, error rate changes
- Close with reflection: lessons, follow-up improvements, prevention steps
